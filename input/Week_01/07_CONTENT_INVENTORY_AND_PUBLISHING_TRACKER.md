**Nguồn tạo:** Kiểm tra từ `06_OUTPUT_WEEK_01_CONTENT_PACKAGE.md`.

---

## 1. Nhận xét sau khi check file 06

File `06_OUTPUT_WEEK_01_CONTENT_PACKAGE.md` là **gói sản xuất nội dung**, không phải file quản lý.

Trong file 06 hiện đã có:

| Nhóm | Số lượng | Ghi chú |
|---|---:|---|
| Video ngắn | 3 | Đủ để đăng 3 ngày |
| Video flagship | 1 | Nên ưu tiên quay kỹ, có consent nếu dùng ảnh trẻ |
| Bài Facebook dài | 1 | Nên đăng ở Facebook cá nhân để xây trust |
| Story | 2 | Dùng để tương tác và kéo lại nội dung chính |
| Backlog tuần sau | 3 | Q3 nám tái phát, Q4 bớt Ota tái phát, Q5 mụn viêm không nên nặn |

**Kết luận:** Tuần 1 đã thừa output để đăng. Cần có tracker riêng để tránh thất lạc nội dung, biết cái nào đã đăng/chưa đăng và chuyển phần dư sang tuần sau.

## 2. Nguyên tắc quản lý kho nội dung

Mỗi nội dung nên đi qua các trạng thái:

```text
Kho ý tưởng
↓
Đã viết nháp
↓
Cần BS duyệt
↓
Sẵn sàng quay
↓
Đã quay
↓
Đã edit
↓
Sẵn sàng đăng
↓
Đã đăng
↓
Review sau 7 ngày
↓
Repurpose nếu tốt
```

Không nên để file output dài như file 06 làm nơi theo dõi trạng thái, vì sau 2-3 tuần sẽ rất khó biết:

- Nội dung nào đã đăng.
- Nội dung nào còn trong kho.
- Nội dung nào cần quay.
- Nội dung nào cần BS Thu duyệt.
- Nội dung nào có thể dùng lại.
- Tuần sau nên lấy gì từ backlog.

## 7. Cột bắt buộc trong file Excel tracker

File Excel đi kèm có các cột:

- Content ID
- Tuần
- Loại
- Pillar/Series
- Chủ đề
- Nguồn input
- Người xem chính
- Insight/Nỗi đau
- Output có sẵn
- Trạng thái
- Ưu tiên
- Kênh chính
- Ngày dự kiến đăng
- Cần ảnh/video?
- Consent/Privacy
- Safety note
- Next action
- Views
- Comments
- Shares
- Saves
- Inbox
- Ghi chú review

Sau khi đăng, chỉ cần cập nhật 6 cột cuối để review:

```text
Views / Comments / Shares / Saves / Inbox / Ghi chú review
```

## 8. Rule sử dụng từ tuần sau

Mỗi tuần nên có 3 file tách riêng:

1. `05_INPUT_WEEK_XX.md`  
   Thu thập case/câu hỏi/hiểu lầm thật.

2. `06_OUTPUT_WEEK_XX_CONTENT_PACKAGE.md`  
   Gói nội dung đã viết để quay/đăng.

3. `07_CONTENT_INVENTORY_TRACKER.xlsx`  
   Kho quản lý trạng thái, lịch đăng, backlog và review số liệu.

Không nên nhét trạng thái đã đăng/chưa đăng vào file 06, vì file 06 nên giữ vai trò là **nội dung thành phẩm**.
