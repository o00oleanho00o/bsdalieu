# README

**Dự án:** Bác Sĩ Thu - Truyền Thông  
**Phiên bản hệ thống:** v1.1  
**Vai trò:** Đây là file AI cần đọc đầu tiên. File này là bản đồ giúp AI biết nên đọc file nào, khi nào, và ưu tiên tài liệu nào khi tạo nội dung cho thương hiệu cá nhân Bác sĩ Thu.

---

# 1. Mục đích của Project

Project này hỗ trợ xây dựng thương hiệu cá nhân **Bác sĩ Thu** trong lĩnh vực da liễu, thẩm mỹ da, laser và chăm sóc sau thủ thuật.

Mục tiêu không phải là bán dịch vụ ngắn hạn.

Mục tiêu chính là xây dựng hình ảnh:

> **Bác sĩ Thu là bác sĩ da liễu có chuyên môn, giải thích dễ hiểu, điều trị có chiến lược, thấu hiểu người bệnh và truyền thông an toàn.**

---

# 2. Nguyên tắc trung tâm

Thương hiệu trung tâm là:

> **Bác sĩ Thu**

Pema chỉ là:
- Bối cảnh chuyên môn.
- Nơi bác sĩ làm việc.
- Điểm chạm thăm khám khi người bệnh cần đánh giá trực tiếp.

Không biến Pema, dịch vụ, máy móc hoặc ưu đãi thành nhân vật chính của nội dung.

Câu định vị lõi:

> **Da khỏe trước, đẹp sau. Điều trị đúng trước, làm đẹp sau.**

Nguyên tắc v1.1:

> **Người xem phải cảm thấy "Bác sĩ hiểu mình" trước khi cảm thấy "Bác sĩ hiểu bệnh."**

---

# 3. Cách đặt tên file trong Project

Project không có folder, vì vậy toàn bộ file nằm ngang. Tên file dùng tiền tố để giả lập nhóm tài liệu:

```text
00_SYSTEM_
01_STRATEGY_
02_CONTENT_
03_PRODUCTION_
04_SAFETY_
05_REFERENCE_
99_ARCHIVE_
```

Ý nghĩa:

| Tiền tố | Vai trò |
|---|---|
| `00_SYSTEM_` | Hướng dẫn hệ thống, custom instructions |
| `01_STRATEGY_` | Định vị, khách hàng, kênh, YouTube |
| `02_CONTENT_` | Trụ cột nội dung, series, cảm xúc, Doctor POV, giọng văn |
| `03_PRODUCTION_` | Quy trình sản xuất video, vận hành nội dung, AI workflow |
| `04_SAFETY_` | Hiểu lầm bệnh nhân, FAQ, kiểm duyệt an toàn y khoa |
| `05_REFERENCE_` | Tài liệu tham khảo, research, benchmark |
| `99_ARCHIVE_` | Bản cũ, không dùng làm nguồn chính |

# 4. Danh sách file v1.1 chính thức

## File hệ thống

### `README.md`

File này.  
AI đọc đầu tiên để hiểu cấu trúc hệ thống.

### `PROJECT_INSTRUCTIONS_PROMPT.md`

---

## File chiến lược

### `01_STRATEGY_BRAND_POSITIONING.md`

Đọc khi cần:
- Định vị thương hiệu.
- Thông điệp lõi.
- Bác sĩ Thu nên được nhớ đến vì điều gì.
- Vai trò của Pema.

### `01_STRATEGY_CUSTOMER_PERSONA.md`

Đọc khi cần:
- Xác định người xem/bệnh nhân mục tiêu.
- Hiểu nỗi đau từng nhóm.
- Chọn insight trước khi viết nội dung.

### `01_STRATEGY_CHANNEL_STRATEGY.md`

Đọc khi cần:
- Chiến lược Facebook cá nhân.
- Chiến lược Fanpage.
- TikTok/Reels.
- Phân phối và tái sử dụng nội dung đa nền tảng.

### `01_STRATEGY_YOUTUBE_STRATEGY.md`

Đọc khi cần:
- YouTube Shorts.
- YouTube long-form.
- Playlist.
- Chuyển video ngắn thắng thành video dài.

---

## File hệ thống nội dung

### `02_CONTENT_CONTENT_PILLARS.md`

Đọc khi cần:
- Lên trụ cột nội dung.
- Lên kế hoạch tuần/tháng.
- Chọn tỷ trọng chủ đề.
- Xác định nội dung thuộc pillar nào.

### `02_CONTENT_SERIES_IDEAS.md`

Đọc khi cần:
- Tạo series video.
- Lên danh sách tập.
- Tổ chức nội dung có tính lặp lại.

### `02_CONTENT_EMOTIONAL_CONTENT_FRAMEWORK.md`

Đọc khi cần:
- Viết hook cảm xúc.
- Chuyển bệnh học thành tình huống đời sống.
- Làm người xem cảm thấy được thấu hiểu.
- Viết mở đầu video/bài Facebook.

### `02_CONTENT_DOCTOR_POV_LIBRARY.md`

Đọc khi cần:
- Tạo nội dung có góc nhìn bác sĩ.
- Viết về điều bác sĩ cân nhắc, từ chối, ưu tiên.
- Tạo authority cá nhân cho Bác sĩ Thu.

### `02_CONTENT_BRAND_VOICE_GUIDE.md`

Đọc khi cần:
- Kiểm soát giọng văn.
- Chỉnh câu chữ.
- Viết caption.
- Viết lời thoại trước camera.

### `02_CONTENT_CONTENT_SCORING_SYSTEM.md`

Đọc khi cần:
- Chấm điểm ý tưởng.
- Chọn video nên quay trước.
- So sánh nhiều ý tưởng.
- Loại ý tưởng không hợp định vị.

- Tái sử dụng nội dung.

Đọc khi cần:
- Dùng AI để nghiên cứu, viết, chỉnh, kiểm duyệt.

- Tối ưu quy trình AI hỗ trợ Bác sĩ Thu.

# 6. Nguyên tắc an toàn bắt buộc

Luôn tránh:
- Cam kết kết quả.
- Quảng cáo lộ liễu.
- Hạ thấp bác sĩ/phòng khám khác.

Thay bằng:
- Có thể cải thiện tùy từng trường hợp.
- Cần đánh giá nền da.
- Cần phân loại tổn thương.
- Cần thăm khám trực tiếp.
- Cần đặt kỳ vọng thực tế.
- Không phải ai cũng phù hợp.

CTA ưu tiên:

> Nếu bạn gặp tình trạng tương tự, nên thăm khám với bác sĩ da liễu để được đánh giá đúng.

# 7. Quy tắc ưu tiên khi có xung đột giữa file

Nếu có nhiều file nói về cùng một chủ đề, ưu tiên:
1. Nội dung trong mục **Hướng Dẫn** của Project.
2. `README.md`.
3. `00_SYSTEM_PROJECT_CUSTOM_INSTRUCTIONS.md`.
4. Các file v1.1 có tiền tố rõ ràng.
5. File cũ chỉ dùng tham khảo nếu không có file mới.

Không ưu tiên:
- File zip.
- File rút gọn cũ.
- File audit cũ.
- File có tên tiếng Việt dài chưa chuẩn hóa.
- File reference chứa thông số/phác đồ nội bộ, trừ khi người dùng yêu cầu rõ.

# 8. Nguyên tắc cuối cùng

Mỗi nội dung phải giúp người xem nghĩ:

> **"Bác sĩ này hiểu vấn đề của mình, nói rõ ràng, không hứa quá, và có trách nhiệm."**

Nếu nội dung chỉ đúng chuyên môn nhưng chưa chạm người bệnh, hãy viết lại theo framework emotion-first.

Nếu nội dung biến Bác sĩ Thu thành người bán dịch vụ, hãy đưa lại về góc nhìn bác sĩ.
